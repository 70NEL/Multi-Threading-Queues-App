# Multi-Threading Queue Simulation

A Java application designed to simulate real-time queue management using advanced multi-threading techniques.

## ⚙️ The Engineering (What I Built)
* **Concurrency:** Implemented thread-safe data structures to manage multiple queues simultaneously.
* **Synchronization:** Used `BlockingQueues` to ensure data integrity between threads.
* **Data Integrity** Made sure that the data used is accurate.

## 🎨 The Simulation (AI-Assisted UI)
* ** To make the data processing visible, I utilized AI tools to generate the boilerplate code for the animated GUI. This allowed for a more sophisticated visualization of the algorithm than a manual build would have permitted within the project's scope. This approach enabled me to:
* ** Prioritize Complex Logic: I dedicated my manual development time exclusively to the back-end concurrency logic and thread-safety, ensuring the core of the application was robust and high-performing.
* ** Enhance Algorithmic Visualization: By leveraging AI for the UI components, I was able to implement a more detailed visual feedback loop, making the behavior of the multi-threading simulation transparent and easier to debug.
* ** Streamline Integration: I focused on the technical challenge of bridging AI-generated interface elements iwith a custom-built, logic-heavy backend, ensuring seamless data flow between the two.
